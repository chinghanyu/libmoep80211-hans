typedef int(* sig_handler) (void)

a signal handler

A sig_handler() is a function pointer that is called by moep_run() after an interrupt.

Returns
The signal handler must either return 0, or moep_run() returns with this return value.

synchronous I/O multiplexing

The moep_select() call allows a program to monitor multiple file descriptors, waiting until one or more of the file descriptors become "ready" for some class of I/O operation. The call works similiar to pselect() and the parameters are identical. See the respective manual page for more details. The difference to pselect() is, that moep_select() schedules the internal moep devices during waiting.

Returns
The return values are the same as for pselect().
Errors
The error values are the same as for pselect().
int moep_run ( sig_handler  sigh)

run the moep multiplexer

The moep_run() call is similiar to calling moep_select() with mostly NULLs in a loop. This call is useful if no additional file descriptors need to be watched. If a signal occured this call does not return but calls sigh instead. If sigh is NULL signals are ignored.

Parameters
sighthe signal handler
Returns
This call does not return until an error occured.
Return values
-1on error, errno is set appropriately.
Errors
The error values are the same as for moep_select(), except that EINTR is handled internally.
